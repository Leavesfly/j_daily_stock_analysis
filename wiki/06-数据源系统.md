# 数据源系统

## 概述

系统实现了 11 种数据源适配器，通过 `DataFetcherManager` 统一管理，支持：

- **自动故障切换**: 当主数据源不可用时自动尝试备选数据源
- **熔断器机制**: 对频繁失败的数据源进行熔断隔离
- **自动恢复**: 熔断后自动进入半开状态探测恢复
- **市场路由**: 根据股票代码自动选择适合的数据源

## 架构设计

```
DataFetcherManager (管理器)
    │
    ├── CircuitBreaker (熔断器)
    │     ├── 失败计数
    │     ├── 熔断阈值
    │     └── 恢复探测
    │
    └── BaseDataFetcher (抽象基类)
          ├── EFinanceFetcher      (A股)
          ├── AkShareFetcher       (A股/港股)
          ├── TushareFetcher       (A股)
          ├── TencentFetcher       (A股/港股)
          ├── BaoStockFetcher      (A股)
          ├── PytdxFetcher         (A股)
          ├── YFinanceFetcher      (美股)
          ├── FinnhubFetcher       (美股/全球)
          ├── AlphaVantageFetcher  (美股/全球)
          ├── LongbridgeFetcher    (港股/美股)
          └── TickFlowFetcher      (A股)
```

## 数据源详情

### A 股数据源

| 数据源 | 说明 | 需要配置 | 优先级 |
|--------|------|---------|--------|
| EFinance | 东方财富数据 | 无 | 高 |
| AkShare | AkShare 开源库 | 无 | 中 |
| Tushare | Tushare Pro | `TUSHARE_TOKEN` | 高（有Token时） |
| Tencent | 腾讯财经 | 无 | 中 |
| BaoStock | 新浪财经等效 | 无 | 低 |
| PytdxFetcher | 通达信协议(TCP+HTTP降级) | 无 | 中 |
| TickFlow | TickFlow API | `TICKFLOW_API_KEY` | 中 |

### 港股/美股数据源

| 数据源 | 适用市场 | 需要配置 |
|--------|---------|---------|
| YFinance | 美股 | 无 |
| Finnhub | 美股/全球 | `FINNHUB_API_KEY` |
| AlphaVantage | 美股/全球 | `ALPHAVANTAGE_API_KEY` |
| Longbridge | 港股/美股 | `LONGBRIDGE_*` 三项 |

## 核心接口

每个数据源实现以下接口（`BaseDataFetcher`）:

```java
public abstract class BaseDataFetcher {
    // 获取历史K线数据
    abstract List<StockDailyData> getHistoryData(String stockCode, LocalDate start, LocalDate end);
    
    // 获取实时行情
    abstract Map<String, Object> getRealtimeQuote(String stockCode);
    
    // 获取股票基本信息
    abstract Map<String, Object> getStockInfo(String stockCode);
    
    // 获取所属板块
    abstract List<String> getStockBoards(String stockCode);
    
    // 数据源名称
    abstract String getName();
    
    // 优先级（数字越小优先级越高）
    abstract int getPriority();
}
```

## 熔断器机制

### 工作原理

```
正常 → [连续失败 >= 阈值] → 熔断(Open)
                                  │
                     [到达恢复时间]↓
                              半开(HalfOpen)
                                  │
                    ┌─────────────┴─────────────┐
               [请求成功]                   [请求失败]
                    │                           │
                    ▼                           ▼
                正常(Closed)               熔断(Open)
```

### 参数说明

| 参数 | 值 | 说明 |
|------|---|------|
| 失败阈值 | 3 次 | 连续失败 3 次触发熔断 |
| 恢复时间 | 60 秒 | 熔断 60 秒后进入半开状态 |
| 半开探测 | 1 次 | 半开状态下发 1 次请求探测 |

## 市场路由规则

系统根据股票代码前缀自动识别市场类型并选择合适的数据源：

| 代码格式 | 市场 | 示例 |
|---------|------|------|
| `6xxxxx` | A 股（上交所） | 600519 |
| `0xxxxx` / `3xxxxx` | A 股（深交所） | 002594, 300750 |
| `8xxxxx` / `4xxxxx` | A 股（北交所） | 830799 |
| `hkxxxxx` | 港股 | hk00700 |
| 纯字母 | 美股 | AAPL, TSLA |

## 数据模型

`StockDailyData` 日K线数据结构:

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| stockName | String | 股票名称 |
| tradeDate | LocalDate | 交易日期 |
| openPrice | Double | 开盘价 |
| highPrice | Double | 最高价 |
| lowPrice | Double | 最低价 |
| closePrice | Double | 收盘价 |
| volume | Long | 成交量 |
| changePct | Double | 涨跌幅(%) |
| dataSource | String | 数据来源标识 |

## 配置示例

```properties
# 使用自动模式（推荐）
DATA_PROVIDER=auto

# 或指定首选数据源
DATA_PROVIDER=efinance

# 配置 Tushare（可选，配置后优先级提升）
TUSHARE_TOKEN=your-token

# 美股需要至少配置一个
FINNHUB_API_KEY=your-key
```
