# 通知系统

## 概述

系统支持 13 种通知渠道，通过 `NotificationRouter` 进行智能路由和降噪控制，确保用户能够及时收到关键分析结果，同时避免信息过载。

## 架构

```
NotificationService (入口)
    │
    ├── NotificationRouter (路由与降噪)
    │     ├── 频率限制 (每小时最大20条)
    │     ├── 去重过滤 (30分钟内相同内容不重发)
    │     ├── 渠道能力协商 (Markdown/图片/长文本)
    │     └── 内容自适应截断
    │
    └── sender/ (渠道发送器)
          ├── WecomSender         (企业微信)
          ├── FeishuSender        (飞书)
          ├── TelegramSender      (Telegram)
          ├── DiscordSender       (Discord)
          ├── SlackSender         (Slack)
          ├── EmailSender         (邮件)
          ├── PushPlusSender      (PushPlus)
          ├── PushoverSender      (Pushover)
          ├── NtfySender          (ntfy)
          ├── GotifySender        (Gotify)
          ├── ServerChan3Sender   (Server酱3)
          ├── CustomWebhookSender (自定义Webhook)
          └── AstrBotSender       (AstrBot)
```

## 降噪机制

### 频率限制

- 每小时最大通知数: **20 条**
- 超过限制时丢弃后续通知，并记录警告日志

### 去重过滤

- 相同内容（标题 + 正文的 MD5 哈希）在 **30 分钟** 内不重复发送
- 自动清理超过 2 小时的历史记录
- 最多保留 100 条去重记录

### 消息路由

根据消息类型自动选择适合的渠道:

| 消息类型 | 路由策略 |
|---------|---------|
| `report` | 所有已配置渠道 |
| `alert` | 所有已配置渠道 |
| `image` | 仅支持图片的渠道（Telegram/Discord/邮件/飞书） |
| `long_report` | 仅支持长文本的渠道（maxLength > 4000） |

## 内容自适应

当消息超过渠道长度限制时，自动截断并添加提示:

```
[消息内容前N字符]

...(内容已截断，完整报告请查看系统)
```

## 通知触发场景

| 场景 | 触发条件 | 内容 |
|------|---------|------|
| 单股分析完成 | 信号非 neutral | 信号、评分、价格、摘要 |
| 批量分析汇总 | 全部分析完成 | 汇总报告 |
| 告警触发 | 满足告警规则条件 | 告警详情 |
| 大盘异动 | 大盘显著变化 | 环境摘要 |

## 通知内容格式

单股分析通知示例:

```
🔥 贵州茅台(600519) 分析完成

信号: strong_buy | 评分: 85/100
价格: 1800.50 (+2.35%)
技术面走强，MACD金叉，均线多头排列...
```

信号对应的 Emoji:
| 信号 | Emoji |
|------|-------|
| strong_buy | 🔥 |
| buy | 📈 |
| sell | 📉 |
| strong_sell | ⚠️ |
| neutral | ⚖️ |

## 配置示例

```properties
# 启用多个渠道
NOTIFICATION_CHANNELS=wecom,telegram,email

# 企业微信
WECOM_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx

# Telegram
TELEGRAM_BOT_TOKEN=123456:ABC-DEF
TELEGRAM_CHAT_ID=-1001234567890

# 邮件
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=your@gmail.com
SMTP_PASSWORD=app-password
EMAIL_TO=receiver@example.com
```

## 自定义 Webhook

支持将通知发送到任意 Webhook 地址:

```properties
CUSTOM_WEBHOOK_URL=https://your-server.com/webhook
CUSTOM_WEBHOOK_METHOD=POST
```

请求体格式:
```json
{
  "title": "通知标题",
  "content": "通知内容",
  "timestamp": "2024-01-01T10:00:00"
}
```
