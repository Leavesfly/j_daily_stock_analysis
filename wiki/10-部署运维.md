# 部署运维

## 部署方式

### 方式一：Docker Compose（推荐）

#### docker-compose.yml

```yaml
version: '3.8'

services:
  dsa-app:
    build: .
    container_name: dsa-java
    ports:
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./data:/app/data    # SQLite数据库持久化
      - ./logs:/app/logs    # 日志持久化
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:8000/api/v1/health"]
      interval: 30s
      timeout: 5s
      retries: 3
```

#### 启动

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down
```

### 方式二：JAR 部署

```bash
# 1. 构建
mvn clean package -DskipTests

# 2. 上传到服务器
scp target/daily-stock-analysis-1.0.0.jar user@server:/opt/dsa/

# 3. 创建 systemd 服务（Linux）
sudo vim /etc/systemd/system/dsa.service
```

#### systemd 服务文件

```ini
[Unit]
Description=Daily Stock Analysis
After=network.target

[Service]
Type=simple
User=dsa
WorkingDirectory=/opt/dsa
ExecStart=/usr/bin/java -jar daily-stock-analysis-1.0.0.jar --serve
EnvironmentFile=/opt/dsa/.env
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable dsa
sudo systemctl start dsa

# 查看状态
sudo systemctl status dsa
```

### 方式三：直接运行

```bash
# 前台运行
java -jar daily-stock-analysis-1.0.0.jar --serve

# 后台运行（nohup）
nohup java -jar daily-stock-analysis-1.0.0.jar --serve > /dev/null 2>&1 &
```

## Dockerfile

```dockerfile
FROM eclipse-temurin:17-jre-alpine

WORKDIR /app
COPY target/daily-stock-analysis-1.0.0.jar app.jar

# 创建数据目录
RUN mkdir -p /app/data /app/logs

EXPOSE 8000

ENTRYPOINT ["java", "-jar", "app.jar", "--serve"]
```

## 数据持久化

### SQLite 数据库

- 路径: `./data/stock_analysis.db`
- 自动创建表结构（JPA ddl-auto: update）
- 包含表: 分析报告、决策信号、投资组合、告警规则、回测记录

### 数据备份

```bash
# 备份数据库
cp ./data/stock_analysis.db ./backup/stock_analysis_$(date +%Y%m%d).db

# 定时备份（crontab）
0 2 * * * cp /opt/dsa/data/stock_analysis.db /opt/backup/dsa_$(date +\%Y\%m\%d).db
```

## 监控

### 健康检查

```bash
# HTTP 健康检查
curl -f http://localhost:8000/api/v1/health || echo "Service Down"
```

### 日志

日志输出格式:
```
HH:mm:ss.SSS [thread] LEVEL logger - message
```

日志级别配置:
```yaml
logging:
  level:
    root: INFO
    com.stock: INFO          # 业务日志
    org.hibernate.SQL: WARN  # 减少SQL日志
```

### 关键监控指标

| 指标 | 检查方式 |
|------|---------|
| 服务存活 | `GET /api/v1/health` |
| 分析次数 | `GET /api/v1/usage` → total_analyses |
| 数据库大小 | `ls -lh ./data/stock_analysis.db` |
| JVM 内存 | `GET /api/v1/system` |

## 运维操作

### 更新部署

```bash
# Docker 方式
docker-compose down
docker-compose build
docker-compose up -d

# JAR 方式
sudo systemctl stop dsa
cp new-version.jar /opt/dsa/daily-stock-analysis-1.0.0.jar
sudo systemctl start dsa
```

### 数据清理

数据库会随分析次数增长，建议定期清理过旧数据:

```bash
# 通过 API 删除旧报告
curl -X DELETE http://localhost:8000/api/v1/analysis/{id}
```

### 故障排查

| 问题 | 排查步骤 |
|------|---------|
| 服务无法启动 | 检查端口占用、Java 版本、.env 文件 |
| 分析超时 | 检查网络（数据源/LLM API）、增大超时时间 |
| 数据源全部失败 | 检查网络连通性、查看熔断器状态 |
| LLM 调用失败 | 验证 API Key、检查余额、确认 API Base URL |
| 通知发送失败 | 检查 Webhook URL、Token 有效性 |
| 内存不足 | 增加 JVM 内存: `java -Xmx512m -jar app.jar` |

### JVM 调优

```bash
# 推荐生产环境配置
java -Xms256m -Xmx512m \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -jar daily-stock-analysis-1.0.0.jar --serve
```

## 安全建议

1. **启用认证**: 生产环境设置 `AUTH_ENABLED=true` 并配置强密码
2. **限制网络**: 通过 Nginx/防火墙限制对外访问端口
3. **保护密钥**: 不要将 `.env` 文件提交到版本库
4. **定期更新**: 保持依赖库和基础镜像的安全更新
5. **日志脱敏**: 确保日志中不会输出 API Key 等敏感信息

## Nginx 反向代理配置示例

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```
