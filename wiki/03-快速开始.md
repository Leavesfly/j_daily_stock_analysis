# 快速开始

## 环境要求

- **Java 17+** (推荐 Eclipse Temurin 17)
- **Maven 3.8+**
- **网络环境**: 需要访问数据源 API 和 LLM API

## 1. 获取代码

```bash
git clone <repository_url>
cd j_daily_stock_analysis
```

## 2. 配置环境变量

```bash
# 复制示例配置文件
cp .env.example .env

# 编辑配置（至少配置以下两项）
vim .env
```

**必须配置项**:
```properties
# LLM API 密钥（必须）
LLM_API_KEY=sk-your-api-key

# 待分析股票列表（必须）
STOCK_LIST=600519,002594,300750
```

## 3. 构建项目

```bash
mvn clean package -DskipTests
```

构建产物: `target/daily-stock-analysis-1.0.0.jar`

## 4. 启动服务

### 方式一：直接运行 JAR

```bash
# Web 服务模式（默认，启动后访问 http://localhost:8000）
java -jar target/daily-stock-analysis-1.0.0.jar --serve

# 单次分析模式
java -jar target/daily-stock-analysis-1.0.0.jar --stocks 600519,hk00700,AAPL

# 干运行（不调用 LLM，仅测试流程）
java -jar target/daily-stock-analysis-1.0.0.jar --stocks 600519 --dry-run
```

### 方式二：Docker Compose

```bash
docker-compose up -d
```

### 方式三：开发模式

```bash
mvn spring-boot:run
```

## 5. 验证服务

```bash
# 健康检查
curl http://localhost:8000/api/v1/health

# 系统信息
curl http://localhost:8000/api/v1/system

# 触发分析
curl -X POST http://localhost:8000/api/v1/analysis/run \
  -H "Content-Type: application/json" \
  -d '{"stocks": "600519"}'
```

## 6. 访问 WebUI

打开浏览器访问: `http://localhost:8000/web/`

主要页面:
- `/web/` - 仪表盘（分析概览）
- `/web/chat` - AI 问股对话
- `/web/history` - 分析历史
- `/web/settings` - 系统设置

## 排错指南

### 启动失败

1. **端口占用**: 修改 `SERVER_PORT` 环境变量或 `application.yml` 中的端口
2. **无法连接 LLM**: 检查 `LLM_API_KEY` 和 `LLM_API_BASE` 配置
3. **数据源超时**: 确认网络可访问对应数据源，或切换 `DATA_PROVIDER`

### 分析无结果

1. 检查股票代码格式（A 股: `600519`，港股: `hk00700`，美股: `AAPL`）
2. 确认非交易时间数据源可能返回空数据
3. 使用 `--dry-run` 排除 LLM 调用问题
