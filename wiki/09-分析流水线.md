# 分析流水线

## 概述

`StockAnalysisPipeline` 是系统的核心编排器，负责协调多个子系统完成完整的股票分析流程。对应 Python 版本的 `src/core/pipeline.py`（3578 行），Java 版完整还原了所有步骤。

## 完整分析流程

### 批量分析入口 (`runFullAnalysis`)

```
1. 解析股票列表（支持代码/名称混合输入）
2. 加载大盘环境（全局共享，避免重复调用）
3. 并发分析每只股票（线程池，最大4线程）
4. 收集结果（单股超时300秒）
5. 结果聚合 + 通知推送
6. 生成汇总报告
```

### 单股分析 18 步骤 (`analyzeStock`)

| Step | 阶段 | 说明 |
|------|------|------|
| 1 | 获取历史数据 | 通过 DataFetcherManager 自动切换数据源 |
| 2 | 获取实时行情 | 当日实时价格和成交量 |
| 3 | 实时数据注入 | 将实时数据追加为历史数据最后一条 |
| 4 | 技术分析 | TA-Lib 计算 MACD/RSI/布林带/均线等 |
| 5 | 新闻搜索 | 搜索股票相关新闻和公告 |
| 6 | 上下文增强 | 注入板块、情报、大盘环境等 |
| 7 | 持久化情报 | 加载历史情报记录 |
| 8 | LLM/Agent 分析 | AI 综合分析（区分 Agent 模式和传统模式） |
| 9 | 趋势 Fallback | 当 LLM 未给出明确信号时用技术分析兜底 |
| 10 | 决策动作刷新 | 补充操作建议 |
| 11 | 大盘环境护栏 | 大盘极弱时降低买入信号强度 |
| 12 | 价格位置填充 | 计算当前价格在52周区间中的位置 |
| 13 | Dashboard 回填 | 补充置信度、止损价、目标价等字段 |
| 14 | 报告语言标准化 | 标准化信号名称、评分范围 |
| 15 | 构建最终报告 | 生成 AnalysisReport 对象 |
| 16 | 保存数据库 | 持久化报告到 SQLite |
| 17 | 提取决策信号 | 从报告中提取并持久化决策信号 |
| 18 | 刷新诊断快照 | 记录耗时和执行状态 |

## 上下文增强链

在发送给 LLM/Agent 之前，系统会构建丰富的分析上下文:

```
基础上下文
├── stock_code          # 股票代码
├── stock_name          # 股票名称
├── market              # 所属市场
├── analysis_date       # 分析日期
├── history_data        # 近20日K线（格式化为表格）
├── realtime_quote      # 实时行情
├── technical_analysis  # 技术分析结果
└── news                # 相关新闻

增强上下文
├── market_context      # 大盘环境
├── market_sentiment    # 市场情绪
├── belong_boards       # 所属板块
├── volume_desc         # 量价关系描述
├── ma_status           # 均线状态
└── intelligence        # 持久化情报
```

### 量价关系描述规则

| 量比 | 描述 |
|------|------|
| > 3.0 | 极度放量（X.X倍） |
| > 2.0 | 显著放量（X.X倍） |
| > 1.5 | 温和放量 |
| > 0.7 | 量能平稳 |
| <= 0.7 | 明显缩量（X.X倍） |

### 均线状态判断

| 条件 | 描述 |
|------|------|
| 收盘 > MA5 > MA10 > MA20 | 多头排列 |
| 收盘 < MA5 < MA10 < MA20 | 空头排列 |
| 其他 | 均线交织 |

## 结果后处理链

### 趋势 Fallback

当 LLM 返回中性评分（50分）或无信号时：
- 评分回退到技术分析趋势评分
- 信号回退: 评分 >= 70 → buy, 评分 <= 30 → sell

### 大盘环境护栏

当大盘情绪为"谨慎"时:
- `strong_buy` → 降级为 `buy`
- 追加风险备注

### Dashboard 字段回填

| 字段 | 回填规则 |
|------|---------|
| 置信度 | 评分非50 → "中等"，评分=50 → "低" |
| 止损价 | 买入信号: 当前价 × 0.92；其他: 当前价 × 0.95 |
| 目标价 | 买入信号: 当前价 × 1.15；其他: 当前价 × 1.08 |
| 摘要 | `"{股票名} 综合评分{X}/100，信号: {signal}"` |

## 进度回调

Pipeline 支持注册进度回调，用于前端进度展示:

```java
pipeline.setProgressCallback((progress, message) -> {
    // progress: 0-100
    // message: 当前步骤描述
    webSocket.send(progress, message);
});
```

进度节点:
| 进度 | 事件 |
|------|------|
| 5% | 股票列表解析完成 |
| 10% | 大盘环境加载完成 |
| 10~80% | 单股分析进度（线性分配） |
| 85% | 分析完成，准备推送 |
| 100% | 全部完成 |

## 诊断系统

每次分析都会记录详细的诊断信息:

```java
DiagnosticContext diag = new DiagnosticContext(stockCode);
diag.stage("fetch_history");      // 标记阶段
diag.record("history_count", N);  // 记录数据
diag.fail("error message");       // 记录失败
diag.complete(elapsed);           // 标记完成
```

## AnalysisResult 中间模型

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| stockName | String | 股票名称 |
| signal | String | 操作信号 |
| score | Integer | 综合评分 (0-100) |
| fullReport | String | 完整分析报告文本 |
| summary | String | 分析摘要 |
| operationAdvice | String | 操作建议 |
| confidence | String | 置信度 |
| riskNote | String | 风险备注 |
| trendLabel | String | 趋势标签 |
| currentPrice | Double | 当前价格 |
| targetPrice | Double | 目标价格 |
| stopLossPrice | Double | 止损价格 |
| pricePosition | Double | 价格位置(0-100) |
| source | String | 分析来源标识 |
