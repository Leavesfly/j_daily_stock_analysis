# 系统架构

## 整体架构

系统采用分层架构设计，核心流程围绕**分析流水线 (Pipeline)** 展开：

```
┌─────────────────────────────────────────────────────┐
│                    接入层                             │
│  REST API (13 Controllers) │ WebUI │ Bot │ 命令行    │
└────────────────────────────┬────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────┐
│                    核心编排层                         │
│  StockAnalysisPipeline │ AgentOrchestrator          │
│  AnalysisContextBuilder │ AnalysisResultAggregator  │
└────────────────────────────┬────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────┐
│                    服务层                             │
│  TechnicalAnalysis │ Intelligence │ MarketContext    │
│  DecisionSignal │ Portfolio │ Alert │ Backtest       │
└────┬──────────────────┬────────────────────┬────────┘
     │                  │                    │
┌────▼────┐    ┌───────▼────────┐    ┌─────▼──────┐
│ 数据层   │    │  AI/LLM 层     │    │  通知层     │
│ 11数据源 │    │  LlmService    │    │  13种渠道   │
│ 自动切换  │    │  6类Agent      │    │  路由+降噪  │
│ 熔断恢复  │    │  Memory+Skills │    │  能力协商   │
└────┬────┘    └───────┬────────┘    └─────┬──────┘
     │                  │                    │
┌────▼──────────────────▼────────────────────▼────────┐
│                   基础设施层                          │
│  SQLite (JPA) │ OkHttp │ Quartz │ JWT │ Config     │
└─────────────────────────────────────────────────────┘
```

## 模块说明

### 1. 接入层 (api / web / bot)

| 模块 | 职责 |
|------|------|
| `api/controller/` | 13 个 REST API Controller，提供完整的对外接口 |
| `web/` | Thymeleaf 页面控制器，提供 WebUI |
| `bot/` | Bot 平台集成（命令解析、平台适配） |
| `Application.java` | 命令行入口，支持 6 种运行模式 |

### 2. 核心编排层 (core)

| 类 | 职责 |
|----|------|
| `StockAnalysisPipeline` | 分析流水线主控，编排 18+ 步骤的完整分析流程 |
| `AnalysisContextBuilder` | 构建分析上下文，整合多维数据 |
| `AnalysisResultAggregator` | 结果聚合器，汇总多股分析结果并推送通知 |
| `AnalysisGuardrails` | 分析护栏，防止异常结果输出 |
| `MarketEngine` | 行情引擎 |
| `TradingCalendar` | 交易日历（判断交易时间） |

### 3. AI Agent 系统 (agent)

| 类 | 职责 |
|----|------|
| `AgentOrchestrator` | Agent 编排器，协调多 Agent 协作 |
| `BaseAgent` | Agent 基类 |
| `TechnicalAgent` | 技术分析 Agent |
| `IntelAgent` | 情报分析 Agent |
| `DecisionAgent` | 决策 Agent |
| `RiskAgent` | 风险评估 Agent |
| `PortfolioAgent` | 投资组合 Agent |
| `AgentMemory` | Agent 记忆系统 |
| `AgentRuntime` | Agent 运行时环境 |
| `LlmToolAdapter` | LLM 工具适配器 |

### 4. 数据源系统 (dataprovider)

| 类 | 职责 |
|----|------|
| `DataFetcherManager` | 数据源管理器（策略模式 + 熔断器） |
| `BaseDataFetcher` | 数据源基类（抽象接口） |
| 11 个 Fetcher 实现 | EFinance/AkShare/Tushare/Tencent/BaoStock/PytdxFetcher/YFinance/Finnhub/AlphaVantage/Longbridge/TickFlow |

### 5. 通知系统 (notification)

| 类 | 职责 |
|----|------|
| `NotificationService` | 通知服务主入口 |
| `NotificationRouter` | 路由 + 降噪 + 能力协商 |
| `sender/` 目录 | 13 种渠道发送器实现 |

### 6. 服务层 (service)

| 服务 | 职责 |
|------|------|
| `TechnicalAnalysisService` | 技术分析计算（TA-Lib） |
| `IntelligenceService` | 智能情报服务 |
| `DailyMarketContextService` | 每日大盘环境构建 |
| `DecisionSignalService` | 决策信号提取与持久化 |
| `PortfolioService` | 投资组合管理 |
| `AlertService` / `AlertWorker` | 告警规则管理与检测 |
| `BacktestService` | 策略回测引擎 |
| `NewsSearchService` | 新闻搜索服务 |
| `MarketAnalysisService` | 大盘分析服务 |
| `HistoryLoaderService` | 历史数据加载 |

## 设计模式

| 模式 | 应用场景 |
|------|---------|
| 策略模式 | 数据源管理器（多数据源自动切换） |
| 模板方法 | BaseDataFetcher / BaseAgent |
| 观察者模式 | 进度回调机制 |
| 责任链模式 | 分析结果后处理链（Fallback → 护栏 → 回填） |
| 熔断器模式 | 数据源故障隔离与自动恢复 |
| 编排器模式 | AgentOrchestrator 多 Agent 协作 |
| 工厂模式 | 通知渠道创建 |

## 数据流

```
用户请求/定时触发
    │
    ▼
解析股票列表 → 加载大盘环境（全局共享）
    │
    ▼ (并发)
┌─────────────── 单股分析循环 ───────────────┐
│  获取历史数据 → 获取实时行情 → 注入实时数据   │
│      │                                      │
│      ▼                                      │
│  技术分析 → 新闻搜索 → 上下文增强            │
│      │                                      │
│      ▼                                      │
│  LLM/Agent 分析 → Fallback → 护栏 → 回填    │
│      │                                      │
│      ▼                                      │
│  构建报告 → 保存数据库 → 提取决策信号         │
└──────────────────────────────────────────────┘
    │
    ▼
结果聚合 → 通知推送
```
